查看使用说明
在 AJAX 请求中，contentType 属性指定了发送到服务器的数据类型。以下是一些常用的 contentType 值：

application/x-www-form-urlencoded：
这是 POST 请求的默认内容类型。
数据以键值对的形式编码，每对之间用 & 分隔，每个键和值之间用 = 分隔。
示例：key1=value1&key2=value2

multipart/form-data：
当你需要上传文件和其他表单数据时使用。
数据被分割成多个部分，每部分包含一块数据，用边界字符串分隔。
常用于包含文件的表单提交。

application/json：
数据以 JSON 格式发送。
适用于发送复杂的数据结构，如对象或数组。
示例：{"key1":"value1","key2":"value2"}

text/plain：
数据以纯文本形式发送。
适用于发送简单的文本数据，无需编码。
示例：Some plain text data

application/xml：
数据以 XML 格式发送。
用于发送 XML 数据。
示例：<root><key1>value1</key1><key2>value2</key2></root>

text/html：
数据以 HTML 格式发送。
可用于发送 HTML 格式化的数据。
示例：<div><p>Some HTML data</p></div>
